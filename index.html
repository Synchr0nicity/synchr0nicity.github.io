Cascading Style Sheets, and their Flow

By Dominic N. Abbott

Hierarchy 👑

CSS stands for "Cascading Style Sheets." Like water that trickles down, from one rock to the next, CSS Selectors affect every element on the page according to a hierarchy. If you were to use selectors simply like water trickling down a river, in top-down fashion, you would have a hard time keeping your styling from running through each element on the page. In order to have a smooth-flowing webpage, the page needs to be planned out in sections, so that each Selector is affecting the bare minimum elements required.

Grouping 🍱

On the other hand, when dealing with styles that stay the same throughout the document- you will be able to keep things clean and easy to read/edit by grouping the same style affects that effect different selectors.

CSS Selectors 💢

Below you will find a table which has the different selectors that must be used to differentiate your style affects:

Name Description Examples
Element Selector Selects HTML elements based on the element name itself p {...}
ID Selector Selects the given id attribute of an HTML element #element {...}
Class Selector The class selector selects HTML elements with a specific class attribute .selectors {...}
Universal Selector Selects all html elements on the page * {...}
Elements can also be combined. For Example: '.selector + p' or '#example + h2' or 'div + p'
